# QazPostWeb — План сборки фронта с ИИ-агентом (по шагам)

Как пользоваться: положи рядом `FRONTEND_BRIEF_FULL.md` (в корень репозитория)
и `design-reference.html` (в папку `design/`). Дальше отправляй агенту (Claude
Code в VS Code) сообщения по очереди: сперва стартовое, потом по одному шагу.
После каждого шага агент показывает результат и останавливается — ты проверяешь
и пишешь «дальше».

## Стартовое сообщение (отправить первым)

Будем собирать фронтенд для backend QazPostWeb в папке `frontend/`. Контракт,
роли и порядок — в `FRONTEND_BRIEF_FULL.md` (единственный источник правды).
Визуальный язык — в `design/design-reference.html`.

Правила работы, соблюдай строго:
1. Делай ТОЛЬКО тот шаг, который я прошу. После него покажи, что сделал, и
   остановись. Не забегай вперёд, не создавай страницы, которые я не просил.
2. Жёсткие требования (раздел 3 и 7 брифа): логин — POST /api/auth/login телом
   form-urlencoded (НЕ JSON); Authorization: Bearer на каждый /api запрос; роль
   из /auth/me; 401 -> /login, 403 -> «нет доступа»; generated_by/printed_by не
   слать; PDF как blob; department_id резолвить в имя через дерево; типы только
   из /openapi.json.
3. Навигацию и роуты гейти по ролям из раздела 6 брифа.

Сейчас сделай только Шаг 1: каркас (Vite + React + TS + Tailwind + React Router
+ react-i18next ru/kz + TanStack Query), зашей дизайн-токены из раздела 4, собери
AppShell с сайдбаром (пункты фильтруются ролью), переключателем РУ/ҚАЗ, бейджем
роли и кнопкой «Выйти». Страницы пока не делай. Покажи результат и остановись.

## Проверка после каждого шага
1. Вид совпадает с design-reference.html.
2. Гард по роли работает — зайди под admin / operator / client.
3. Нет ошибок в консоли и сети (особенно 401/403/422).

Если сломалось — вставь агенту текст ошибки или сетевой ответ, не проси «исправь
вообще». Если расходится с контрактом бэка — сверь по FRONTEND_BRIEF_FULL.md или
спроси отдельно.

## Ролевой доступ (шпаргалка из раздела 6 брифа)
- admin: все страницы.
- operator: всё кроме Пользователи / Аудит / создание-правка Клиентов
  (клиентов только смотрит).
- client: только Генерация, свои Заявки на диапазон, Деталь/предпросмотр своей
  партии. Остальное скрыто (403).
